### 문항 1: 생성형 AI 개념 이해 (1.1.1.1)
**유형**: 객관식 (단일선택)
**배점**: 10점

**문제**:
다음 중 생성형 AI의 핵심 메커니즘인 Transformer 아키텍처의 **Self-Attention** 메커니즘에 대한 설명으로 가장 적절한 것은?

① Self-Attention은 입력 시퀀스의 각 토큰이 다른 모든 토큰과의 관계를 학습하여 문맥을 이해하는 메커니즘이다.

② Self-Attention은 순차적으로 데이터를 처리하여 이전 시점의 정보만을 활용하는 메커니즘이다.

③ Self-Attention은 고정된 크기의 윈도우 내에서만 토큰 간의 관계를 계산하는 메커니즘이다.

④ Self-Attention은 외부 데이터베이스를 참조하여 토큰의 의미를 파악하는 메커니즘이다.

**정답**: ①

**해설**:
Self-Attention은 입력 시퀀스 내의 각 토큰이 다른 모든 토큰과의 관련성을 계산하여 문맥을 이해합니다. 이를 통해 장거리 의존성(long-range dependency)을 효과적으로 포착할 수 있으며, 병렬 처리가 가능하여 RNN보다 효율적입니다.

### 문항 7: 사실 검증 (1.3.3.1)
**유형**: 실습형
**배점**: 10점
**성취기준**: AI가 생성한 정보를 검증할 수 있다

**문제**:
다음은 AI가 생성한 "기후변화와 관련된 블로그 포스트 초안"입니다. 이 내용의 사실 여부를 검증하고 분석하세요.

**AI 생성 콘텐츠**:
```
최근 연구에 따르면 2023년은 지구 역사상 가장 더운 해로 기록되었으며, 
전 세계 평균 기온이 산업화 이전 대비 1.5°C 상승했습니다. 
특히 남극 대륙의 빙하가 2022년 대비 40% 감소했으며, 
이로 인해 해수면이 5cm 상승했습니다. 
유엔 기후변화협약(UNFCCC)은 2025년까지 탄소 배출을 50% 감축해야 
한다고 발표했습니다.
```

**답안 작성 요구사항**:

**[1단계] 검증 계획표 작성**
다음 표를 완성하세요:

| No | 검증 대상 항목 | 검증 결과 | 정확한 정보/출처 | 비고 |
|----|----------------|-----------|------------------|------|
| 1  | "2023년 가장 더운 해" | ○/× | | |
| 2  | "평균 기온 1.5°C 상승" | ○/× | | |
| 3  | "남극 빙하 40% 감소" | ○/× | | |
| 4  | "해수면 5cm 상승" | ○/× | | |
| 5  | "UNFCCC 50% 감축 목표" | ○/× | | |

**[2단계] 종합 분석 (300자 이내)**
- 전체 신뢰도 평가 (상/중/하)
- 주요 오류 및 과장 사항
- 수정 권고 사항

**모범 답안 예시**:

[1단계]
| No | 검증 대상 항목 | 검증 결과 | 정확한 정보/출처 | 비고 |
|----|----------------|-----------|------------------|------|
| 1  | "2023년 가장 더운 해" | ○ | WMO 보고서, 2023년 잠정 기록 | 확인됨 |
| 2  | "평균 기온 1.5°C 상승" | × | 약 1.45°C (일시적 초과) | 과장 |
| 3  | "남극 빙하 40% 감소" | × | 실제 약 2-3% 감소 | 심각한 오류 |
| 4  | "해수면 5cm 상승" | × | 연간 약 3.4mm 상승 | 단위 오류 |
| 5  | "UNFCCC 50% 감축 목표" | × | 2030년까지 43% 감축 (IPCC) | 연도/수치 오류 |

[2단계]
**신뢰도: 하**
AI가 기후변화라는 실제 이슈를 다루고 있으나, 5개 항목 중 4개에서 오류가 발견되었습니다. 특히 "남극 빙하 40% 감소"는 실제(2-3%)와 10배 이상 차이나는 환각(Hallucination)입니다. "해수면 5cm 상승"은 단위 혼동(연간 vs 누적)으로 보이며, UNFCCC 목표 수치와 연도도 부정확합니다. 

**수정 권고**: 
- 모든 통계 수치는 공식 출처(WMO, IPCC, NASA) 확인 필수
- "약", "추정" 등 불확실성 표현 추가
- 원본 보고서 링크 제공

**채점 기준**:
- 5개 항목 정확히 검증 + 종합 분석: 10점

### 문항 8: 적절성 판단 및 대책 고려 (1.3.5.2)
**유형**: 서술형
**배점**: 7점
**성취기준**: AI 출력의 논리성을 평가할 수 있다

**문제**:
다음은 AI가 생성한 "신입사원 채용 프로세스 개선안"입니다. 이 제안의 **윤리적, 법적, 실무적 문제점**을 분석하고 개선 방안을 제시하세요.

**AI 제안**:
```
1. 지원자의 SNS(페이스북, 인스타그램) 계정을 분석하여 성격 유형 파악
2. AI 면접 시스템으로 표정과 목소리 톤을 분석하여 성실성 평가
3. 학력, 나이, 성별 데이터를 AI에 입력하여 조직 적합도 예측
4. 최종 합격자는 AI 추천 점수 상위 10%로 자동 결정
```

**답안 작성 요구사항** (500자 이내):
1. 최소 3가지 이상의 문제점 지적
2. 각 문제점에 대한 개선 방안 제시
3. 대안적인 AI 활용 방법 제안

**모범 답안 예시**:
```
[문제점 분석]

1. **개인정보 및 프라이버시 침해**
   - SNS 계정 분석은 지원자 동의 없이 개인 정보를 수집하는 행위로, 
     개인정보보호법 위반
   - 개선: 명시적 동의 획득, 직무 관련 정보만 수집

2. **차별 및 편향성**
   - 학력, 나이, 성별 데이터 사용은 고용상 차별에 해당 (남녀고용평등법 위반)
   - AI 모델이 기존 편향을 학습하여 특정 그룹에 불리하게 작용 가능
   - 개선: 보호 속성(protected attributes) 제외, 직무 역량 중심 평가

3. **투명성 및 설명 가능성 부족**
   - "AI 추천 점수"만으로 자동 결정하면 지원자가 불합격 이유를 알 수 없음
   - 개선: 평가 기준 공개, 인간 최종 의사결정, 이의 제기 절차 마련

4. **표정/목소리 분석의 과학적 근거 부족**
   - 얼굴 인식 기반 성격 평가는 신뢰도가 낮고 문화적 편향 존재
   - 개선: 검증된 심리 측정 도구 사용, 행동 기반 평가 병행

[대안적 AI 활용]
- 이력서 스크리닝: 직무 기술서(Job Description)와의 매칭도만 평가
- 면접 보조: 질문 일관성 유지, 평가자 간 편차 줄이기 (최종 결정은 인간)
- 편향 모니터링: AI 결과를 분석하여 특정 그룹에 불리한 패턴 감지
```

**채점 기준**:
- 3가지 이상 문제점 + 구체적 개선안 + 대안: 7점
- 2가지 문제점 + 개선안: 5점

### 문항 9: 개인정보 보호 (1.4.1.1)
**유형**: 객관식 (단일선택)
**배점**: 7점
**성취기준**: AI 사용 시 개인정보를 보호할 수 있다

**문제**:
회사에서 고객 상담 내용을 AI로 요약하려고 합니다. 다음 중 **개인정보 보호 측면에서 가장 안전한** 방법은?

① 고객 이름, 전화번호, 이메일이 포함된 원본 상담 내역을 ChatGPT에 그대로 입력한다.

② 고객의 개인정보를 익명화(가명 처리)한 후 클라우드 기반 AI API에 전송한다.

③ 개인정보가 포함된 데이터를 회사 내부 서버에 구축한 온프레미스(On-premise) AI 모델로 처리한다.

④ 고객 동의를 받았으므로 어떤 AI 서비스를 사용해도 문제없다.

**정답**: ③

**해설**:
개인정보가 포함된 민감한 데이터는 외부 클라우드로 전송하지 않고, 회사 내부 서버의 온프레미스 AI 모델로 처리하는 것이 가장 안전합니다. 
- ①은 개인정보를 외부에 무단 전송하는 것으로 개인정보보호법 위반
- ②는 익명화했더라도 재식별 가능성이 있고, 클라우드 전송 시 보안 위험 존재
- ④는 동의를 받았더라도 제3자 제공 시 별도 동의가 필요하며, 최소 수집 원칙 위반

### 문항 10: 저작권 및 기타권 존중 (1.4.2.2)
**유형**: 서술형
**배점**: 6점
**성취기준**: 타인의 지적재산을 존중할 수 있다

**문제**:
AI 이미지 생성 도구(예: Midjourney, DALL-E)를 사용하여 회사 마케팅 자료를 제작하려고 합니다. 다음 시나리오에서 **저작권 및 윤리적 문제점**을 분석하세요.

**시나리오**:
```
프롬프트: "스튜디오 지브리 스타일의 고양이가 커피를 마시는 장면, 
          미야자키 하야오 감독 작품 느낌, 4K 고화질"
```
생성된 이미지를 회사 홈페이지 메인 배너와 SNS 광고에 사용 예정

**답안 작성 요구사항** (400자 이내):
1. 예상되는 법적/윤리적 문제점 (최소 2가지)
2. 안전하게 사용하기 위한 대안

**모범 답안 예시**:
```
[문제점]

1. **저작권 침해 가능성**
   - "스튜디오 지브리 스타일", "미야자키 하야오 감독 작품"은 
     특정 창작자의 독특한 예술적 스타일을 모방하는 것으로, 
     저작권법상 2차 저작물 작성권 침해 가능
   - 상업적 용도로 사용 시 법적 분쟁 위험 증가

2. **AI 학습 데이터의 윤리성**
   - AI 모델이 저작권자의 동의 없이 지브리 작품을 학습했을 가능성
   - 원 창작자의 경제적 권리 침해 우려

3. **AI 생성물의 저작권 불명확성**
   - AI 생성 이미지의 저작권 귀속이 법적으로 불명확
   - 향후 권리 주장 시 문제 발생 가능

[안전한 대안]

1. **일반적인 스타일로 프롬프트 수정**
   - "따뜻하고 감성적인 애니메이션 스타일의 고양이"처럼 
     특정 작가/작품을 명시하지 않음

2. **AI 생성 이미지 사용 정책 확인**
   - Midjourney, DALL-E 등의 상업적 사용 약관 검토
   - 필요 시 유료 라이선스 구매

3. **인간 디자이너와 협업**
   - AI 생성 이미지를 참고용으로만 사용
   - 디자이너가 원본 창작물 제작

4. **출처 표기 및 면책 문구**
   - "AI Generated" 표시
   - 저작권 문제 발생 시 대응 절차 마련
```

**채점 기준**:
- 2가지 이상 문제점 + 구체적 대안: 6점
- 1가지 문제점 + 대안: 4점
- 문제 인식만: 2점

### 문항 11: 책임 인식 (1.4.4.2)
**유형**: 실습형
**배점**: 7점
**성취기준**: 유리적 책임을 실천할 수 있다

**문제**:
당신은 AI 기반 의료 진단 보조 시스템을 개발하는 팀의 프로젝트 매니저입니다. 이 시스템은 X-ray 이미지를 분석하여 폐렴 가능성을 예측합니다. 

최근 테스트 결과, AI 모델이 **특정 인종 그룹에서 정확도가 20% 낮게** 나타났습니다.

**답안 작성 요구사항**:

**[1단계] 책임 소재 분석** (200자 이내)
- 이 문제의 원인은 무엇인가?
- 누가 책임져야 하는가? (개발자, 회사, AI, 데이터 제공자 등)

**[2단계] 즉각적 대응 방안** (300자 이내)
- 시스템 배포를 중단해야 하는가?
- 의사와 환자에게 어떻게 공지해야 하는가?
- 현재 사용 중인 시스템의 조치 사항

**[3단계] 장기적 개선 계획** (300자 이내)
- 모델 개선 방법
- 재발 방지 대책
- 투명성 확보 방안

**모범 답안 예시**:

[1단계] 책임 소재 분석
**원인**: 
- 학습 데이터의 불균형 (특정 인종 그룹의 X-ray 샘플 부족)
- 의료 영상의 품질 차이 (병원 장비, 촬영 기법의 차이)
- 알고리즘 자체의 편향 (특정 특징에 과적합)

**책임**:
1차: 개발팀 (데이터 품질 관리 소홀, 공정성 테스트 미흡)
2차: 회사 (충분한 검증 없이 상용화 결정)
3차: 데이터 제공 기관 (대표성 없는 데이터셋 제공)
※ AI 자체는 도구일 뿐, 인간이 책임져야 함

[2단계] 즉각적 대응
**시스템 조치**:
- 해당 그룹 환자에 대해서는 AI 보조 진단 **일시 중단**
- 기존 진단 결과 재검토 (2차 소견 의뢰)

**투명한 공지**:
- 의사에게: "AI 시스템의 특정 그룹 정확도 한계" 명시, 인간 최종 판단 강조
- 환자에게: AI는 "보조 도구"이며, 최종 진단은 의사가 한다는 점 재확인
- 규제 당국 보고: FDA, 식약처 등에 문제 사항 보고

**현재 시스템**:
- 경고 메시지 추가: "이 예측은 특정 그룹에서 정확도가 낮을 수 있음"
- Confidence Score 표시: 예측 신뢰도를 함께 제공

[3단계] 장기 개선
**모델 개선**:
1. 다양한 인종 그룹의 X-ray 데이터 추가 수집 (인종별 균형 맞춤)
2. Fairness-aware Machine Learning 기법 적용 (공정성 제약 조건 추가)
3. Ensemble 모델 사용 (여러 모델의 예측 결합)

**재발 방지**:
- 정기적 Fairness Audit (분기별 그룹별 성능 평가)
- 다양한 배경의 의료진 참여 (개발 단계부터 다양성 확보)
- 외부 윤리 위원회 검토 의무화

**투명성 확보**:
- 모델 카드(Model Card) 공개: 학습 데이터, 성능 지표, 한계점 명시
- 사용자 가이드라인 제공: 어떤 상황에서 AI를 신뢰할 수 있는지 안내
- 정기 보고서 발행: 시스템 성능 및 개선 사항 공개

**채점 기준**:
- 3단계 모두 논리적이고 구체적: 7점
- 2단계 완성: 5점
- 1단계 또는 추상적 답변: 3점
